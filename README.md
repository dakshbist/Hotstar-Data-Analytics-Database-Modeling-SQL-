# 📺 Hotstar Data Analytics & Database Modeling (SQL Project)

## 📌 Project Overview
This project focuses on designing and analyzing a relational database system inspired by a real-world OTT streaming platform similar to Hotstar. The goal is to simulate how streaming platforms manage and analyze data related to users, content, subscriptions, and engagement.

The project demonstrates strong SQL skills by extracting meaningful insights from structured data using advanced querying techniques.



## 🎯 Objectives
- Design a normalized relational database
- Analyze user behavior and content consumption
- Track subscription trends and revenue growth
- Generate business insights using SQL queries



## 🛠️ Tools & Technologies
- MySQL
- MySQL Workbench
- SQL (Structured Query Language)



## 🗂️ Database Schema

The database consists of the following tables:

- **Users** – Stores user details and subscription tier
- **Content** – Stores movies, series, and sports content
- **Subscriptions** – Tracks payment and subscription status
- **Watch_History** – Records user viewing activity
- **Watchlist** – Stores content saved for later
- **User_Ratings** – Captures user feedback and ratings
- **Ad_Campaigns** – Tracks advertisement performance



## 🔗 Key Concepts Used
- **Primary & Foreign Keys** (Data Integrity)
- **JOINs** (Combining multiple tables)
- **Aggregation Functions** (SUM, COUNT)
- **GROUP BY** (Data summarization)
- **Subqueries** (Nested queries)
- **Window Functions** (RANK, Running Total)



## 📊 Key Insights

- 📈 **Most Watched Content:** ICC World Cup Final  
- 🎯 **Top Category:** Sports (Highest engagement)  
- 👥 **User Distribution:** Balanced across Premium, Super, and Free tiers  
- 💰 **Revenue Growth:** Consistent increase over time using subscription data  
- ⭐ **User Engagement:** Identified high-engagement users based on watch duration
